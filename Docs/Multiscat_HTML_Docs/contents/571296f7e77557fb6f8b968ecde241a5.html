pot*****.in

UMLInterface
FortranMultiscat :: DependancyTree :: pot*****.in

Description

This file describes the potential which is the source of the scattering. The * a 5 digit index as refered to by Multiscat.conf (see startindex and endindex fields). The first 5 lines are from comments only.

The 3d potential needs to be passed in an unusual form: it is a series of slices parallel to the x-y plane. Along each of these slices the Fourier components must be specified. This means the file contains the components of the Fourier transform of the potential, along the x and y directions only.

These are stored as a list of paired numbers, one pair per line. Each pair represents one component, a complex number saved as (real part, imaginary part). The order of the components is detrmined by the "FourierLabels.in" file. This specifies the order of the components based on their x-y (Fourier) indexing. The z ordering is sequential, from bottom to top.

To give an example (copied fro Fay's guide, with minor changes) suppose we would like to express the potential in a 3x3x2 array, that is 3x3 grids evaluated at two z positions (z=0 and z=1). The FourierLabels.in file could look like this: 0 0 -1 0 0 1 ...

The pot*.in would then look like this (here V is the potential Fourier transformed in the x-y plane, written as real, imaginary parts): V( kx = 0, ky = 0, z = 0) V( kx = 0, ky = 0, z = 1) V( kx = -1, ky = 0, z = 0) V( kx = -1, ky = 0, z = 1) V( kx = 0, ky = 1, z = 0) V( kx = 0, ky = 1, z = 1) ...
